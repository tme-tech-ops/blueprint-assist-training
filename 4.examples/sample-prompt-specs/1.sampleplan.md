# Getting Started with Blueprint Assist

## Overview
This example prompt helps a new user get oriented with Blueprint Assist (BPA) on
the Dell Automation Platform (DAP) before building their first blueprint. Adapt
the questions below to your own environment.

## Example Prompt

### 1. Capabilities
What can Blueprint Assist help me do? Give a brief overview of how it supports
writing, linting, validating, and deploying DAP blueprints.

### 2. Sample Blueprints as References
Walk me through how the sample blueprints in this repository are organized, and
which ones are good starting points for a bare-metal OS deployment, storage
provisioning, or a Kubernetes cluster.

### 3. Blueprint Structure
Give me a brief summary of how a DAP blueprint is structured: the main blueprint
file, inputs, capabilities, node templates, and how plugins fit in.

---

## What a Good Answer Covers
- **Capabilities**: the end-to-end blueprint workflow (author, lint, validate, deploy, monitor)
- **References**: where to find relevant sample blueprints and how to reuse them
- **Structure**: the multi-file layout and the role of each blueprint section
- **Next steps**: how to go from a sample blueprint to your own working blueprint

## Usage Notes
- Use this as a first prompt when onboarding to Blueprint Assist
- Adapt the example questions to the platform or workload you care about

# Blueprint Assist Demo Script

- Audience: Advanced users, blueprint developers, plugin contributors
- Topics: Plugin discovery, documentation, fetching, importing custom plugins
- Key Skill: Extending dap-bpa with custom capabilities

## Session Setup

- **Session Type**: Fresh session
- **Bypass Mode**: Enabled

## Notes

- Knowledge plugin commands are local — no orchestrator connection required
- `orchestrator plugins list` (step 8.7) requires DAP credentials and `--trust-all` for dev
- Replace `<orchestrator>` in any command with a profile name returned by `dap-bpa status`
- A custom plugin is optional — steps 8.5 can be shown as a dry-run if none is available

## Demo Steps - Demo 8

### 8.1 Discover available plugins

```bash
dap-bpa knowledge plugins list
```

### 8.2 List the node types for a plugin

```bash
dap-bpa knowledge plugins list vsphere
```

### 8.3 Read a plugin's documentation

```text
Give me an overview of the vsphere plugin — its authentication model, common node types, and typical usage patterns
```

```bash
dap-bpa knowledge plugins docs vsphere
```

### 8.4 Explore a specific node type

```bash
dap-bpa knowledge plugins get vsphere <node-type>
```

```bash
dap-bpa knowledge plugins node-type-docs vsphere <node-type>
```

### 8.5 Fetch / refresh plugin docs

Keep plugin documentation current by fetching the latest.

```bash
# fetch a single plugin
dap-bpa knowledge plugins fetch vsphere

# or refresh all plugins
dap-bpa knowledge plugins fetch --all
```

### 8.6 Import a custom plugin

Add a plugin by name or from a local path so its node types become available to dap-bpa.

```bash
dap-bpa knowledge plugins add <name-or-path>
```

### 8.7 List plugins installed on an orchestrator

Compare the locally known plugins with those available on the target orchestrator.

```bash
dap-bpa orchestrator plugins list --orchestrator <orchestrator> --trust-all
```

## Optional Demo Steps

### 8.8 Search plugin docs for a capability

```bash
dap-bpa knowledge docs search "create virtual machine" --plugin vsphere
```

---

## Notes

- Plugins extend dap-bpa with infrastructure-specific node types
- `knowledge plugins fetch` keeps local plugin docs current
- Custom plugins can be imported with `knowledge plugins add`
- Each plugin provides multiple node types with properties and examples
- Local `knowledge plugins` commands differ from `orchestrator plugins` (which target a live DAP)

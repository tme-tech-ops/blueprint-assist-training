# Blueprint Assist Demo Script

- Audience: DevOps leads, infrastructure teams, orchestrator administrators
- Topics: `dap-bpa setup`, profile configuration, `-o`/`--orchestrator` flag, SSL handling
- Key Skill: Managing multiple orchestrator environments

## Session Setup

- **Session Type**: Fresh session
- **Bypass Mode**: Enabled

## Notes

- This demo can run without an AI session — most steps are direct `dap-bpa` CLI commands
- Profiles are stored in `~/.blueprint-assist/config.json`
- Use `--trust-all` only for dev/self-signed orchestrators — never in production
- Replace `<orchestrator>` in any command with a profile name returned by `dap-bpa status`
- The setup wizard is interactive; have your orchestrator URL and credentials ready

## Demo Steps - Demo 4

### 4.1 Check current setup status

```bash
dap-bpa status
```

### 4.2 Run the interactive setup wizard

Walk through the wizard to register your first orchestrator profile (URL, credentials, default profile name).

```bash
dap-bpa setup
```

### 4.3 Explain the config file structure

```text
Show me where dap-bpa stores its orchestrator profiles and explain the structure of the ~/.blueprint-assist/config.json file
```

### 4.4 List the target orchestrator connections available

```text
Show me the target orchestrator connections that you have available

For self-signed certificates, connect with `--trust-all`
```

### 4.5 Add additional orchestrator profiles

Re-run the wizard to add more environments (for example `staging` and `prod`) so you can switch between them.

```bash
dap-bpa setup
```

### 4.6 Select a profile with the -o / --orchestrator flag

Every orchestrator command accepts a named profile. Show switching between environments without reconfiguring.

```bash
dap-bpa orchestrator blueprints list --orchestrator <orchestrator> --trust-all
```

```bash
dap-bpa orchestrator blueprints list -o <orchestrator> --trust-all
```

### 4.7 Demonstrate SSL certificate handling

Show the difference between a strict connection and a dev connection using `--trust-all` for self-signed certificates.

```bash
dap-bpa orchestrator plugins list --orchestrator <orchestrator> --trust-all
```

Confirm which orchestrator profiles are configured and which one is the default

## Notes

- One-time setup with `dap-bpa setup`; re-run to add or edit profiles
- Profiles let you switch orchestrators without reconfiguring credentials
- `--trust-all` bypasses SSL verification — restrict to development environments
- Verify orchestrator connectivity before running upload or deploy demos

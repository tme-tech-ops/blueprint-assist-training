# Blueprint Assist Demo Script

- Audience: Blueprint authors, architects, trainers, reviewers
- Topics: Blueprint visualization, install-flow execution surface, risk analysis, `.riskignore`, input exploration, composer sub-flows
- Key Skill: Generating and using the interactive HTML visualizer for blueprint review, demos, and documentation

## Session Setup

- **Session Type**: Fresh session
- **Bypass Mode**: Enabled

## Prerequisites

- No DAP orchestrator connection is required — visualization works against local files
- Uses the blueprint generated in Demo 2 at `4.examples/target-build-folder/WindowsVM_Vsphere_demo`
- If that blueprint is not present, run Demo 2 first or point the prompts at any local blueprint
- The `--file` path must be **within the current working directory** — `dap-bpa` rejects paths that resolve outside it
- The generated HTML file is fully self-contained — no server or internet connection required to open it

## Quick Start: How to Run It

**Manual command:**

```bash
dap-bpa blueprint visualize --file ./my-blueprint          # a directory
dap-bpa blueprint visualize --file ./blueprint.yaml        # a single file
dap-bpa blueprint visualize --file ./my-blueprint --output ./custom-name.html
```

Output defaults to `<blueprint-name>-visualizer.html` (derived from the file or directory name) in the current working directory, or the path given by `--output`.

**Automatic generation** — it runs automatically after a successful upload:

```bash
dap-bpa orchestrator blueprints upload --file ./my-blueprint --id my-bp --revision 1.0.0
# ✓ Blueprint visualization generated: my-blueprint-visualizer.html
```

To disable auto-generation, set `BPA_DISABLE_VISUALIZATION=true`. If visualization fails during an upload, the upload itself still succeeds — you can regenerate manually later.

**Common uses:**

- Preview the topology and review risks *before* uploading.
- Sanity-check node/relationship wiring after authoring or editing a blueprint.
- Feed the risk findings into remediation — the related `blueprint-risk-fix` workflow applies fixes from the visualizer / risk report.

## Notes

- The Blueprint Visualizer is a single self-contained HTML file (title: **Blueprint Architecture Visualizer**) — no server or CDN required
- As of v0.28.2 it is a **React app** that replaces the legacy Cytoscape graph; React is inlined and the output contains no `cytoscape`/CDN references
- All blueprint data is embedded as a `BLUEPRINT_DATA` JSON constant (metadata, nodes, relationships, input groups, input details, risks, plugins) in the HTML
- The visualizer is automatically generated as `<blueprint-name>-visualizer.html` when uploading a blueprint to the orchestrator (disable with `BPA_DISABLE_VISUALIZATION=true`)
- Layout: a central **Blueprint Execution Surface** with install-flow animation, an **Infrastructure Map** (platform detection) view, and **Step Details**, **Risk Analysis**, **Plugins**, **Input Groups**, **Sub-Deployment**, and **Source** panels
- **Risk analysis** is static and embedded — computed at generation time from ~25 rules across security, reliability, operability, and lifecycle; each finding links to `docs/RISK_RULES.md`
- **`.riskignore`** (placed next to the blueprint root YAML) suppresses findings by rule id, `rule:node`, or exact title; malformed / no-match patterns produce warnings and the suppressed count is reported
- The install-flow animation is particularly useful for training — it makes TOSCA execution order and parallel vs. sequential steps concrete
- Share the HTML via email, attach to PRs, embed in documentation, or use in demos

# Blueprint Assist Demo Script

- Audience: Blueprint authors, architects, DevOps engineers
- Topics: Blueprint analysis, TOSCA understanding, security & compliance review
- Key Skill: Using AI to understand and improve blueprints

## Session Setup

- **Session Type**: Fresh session
- **Bypass Mode**: Enabled

## Notes

- No DAP orchestrator connection is required — reasoning works against local files
- Uses the blueprint generated in Demo 2 at `4.examples/target-build-folder/WindowsVM_Vsphere_demo`
- If that blueprint is not present, run Demo 2 first or point the prompts at any local blueprint

## Demo Steps - Demo 5

### 5.1 Explain what dap-bpa can analyze

```text
What can dap-bpa analyze and reason about in a blueprint? Explain the kinds of insights it can provide without connecting to an orchestrator
```

### 5.2 Analyze a real blueprint

```text
Analyze the blueprint in 4.examples/target-build-folder/WindowsVM_Vsphere_demo and summarise its node types, inputs, and any issues.
```

### 5.3 Explain the blueprint structure

```text
Walk me through the structure of this blueprint — explain the node types, their relationships, and how the inputs flow into them
```

### 5.4 Explain TOSCA concepts in context

```text
Using this blueprint as an example, explain the TOSCA concepts at play — node templates, relationships, capabilities, and intrinsic functions like get_input and get_secret
```

### 5.5 Security and compliance review

```text
Review this blueprint for security and compliance issues — check secret handling, hardcoded credentials, input validation, and constitution/blueprint-rules compliance
```

### 5.6 Improvement suggestions

```text
Suggest concrete improvements to this blueprint covering structure, reusability, and best practices. Prioritise the highest-impact changes.
```

## Optional Demo Steps

### 5.7 Compare against a knowledge-base example

```text
Find a similar blueprint in the knowledge base and compare it to ours — what patterns does the reference example use that we could adopt?
```

dap-bpa knowledge blueprints find "windows vm vsphere"

# Blueprint Assist Demo Script

- Audience: DevOps engineers, infrastructure teams, automation engineers
- Topics: Monitor Agent workflow, auto-repair, assertions, daemon management
- Key Skill: Automated deployment lifecycle and diagnostics

## Session Setup

- **Session Type**: Fresh session
- **Bypass Mode**: Enabled

## Notes

- Requires a validated blueprint (from Demo 2) and orchestrator access
- Auto-repair requires LLM credentials (AWS Bedrock or OpenAI) configured for the diagnostician
- `--file` expects the blueprint YAML file, not the directory — the blueprint ID is derived from the filename
- `--inputs` expects the JSON inputs file for the blueprint
- On Windows the background daemon `bpa-daemon.exe` lands in `C:\Users\<username>\bin`
- Use `--trust-all` for dev orchestrators with self-signed certificates
- Replace `<orchestrator>` in any command with a profile name returned by `dap-bpa status`

## Demo Steps - Demo 7

### 7.1 Explain the Monitor Agent

```text
What is the dap-bpa Monitor Agent? Explain the full lifecycle it automates and how the LLM-powered diagnostician performs auto-repair
```

### 7.2 Run the full lifecycle (deploy and monitor)

Deploy straight from a local blueprint file and monitor end-to-end until the deployment completes.

```bash
dap-bpa monitor --file 4.examples/target-build-folder/WindowsVM_Vsphere_demo/blueprint.yaml --inputs "4.examples/target-build-folder/WindowsVM_Vsphere_demo/example_JSON_configs/demo3-json.json" --orchestrator <orchestrator> --trust-all
```

### 7.3 Monitor an existing deployment

Start the install workflow on an already-created deployment and monitor it.

```bash
dap-bpa monitor --deployment-id my-windows-vm-001 --orchestrator <orchestrator> --trust-all
```

### 7.4 Validate outputs with assertions

Use `--assert` to validate deployment outputs on completion.

```bash
dap-bpa monitor --deployment-id my-windows-vm-001 --assert "deployment_status == completed" --orchestrator <orchestrator> --trust-all
```

### 7.5 Run in detached (background) mode

```bash
dap-bpa monitor --deployment-id my-windows-vm-001 --detach --orchestrator <orchestrator> --trust-all
```

### 7.6 Check session status

```bash
# most recent session
dap-bpa monitor --status --orchestrator <orchestrator> --trust-all

# a specific session/execution
dap-bpa monitor --status --execution-id <exec-id> --orchestrator <orchestrator> --trust-all
```

### 7.7 Manage the background daemon

```bash
# is the background daemon alive?
dap-bpa monitor --daemon-status

# stop the background daemon when done
dap-bpa monitor --daemon-stop
```

## Notes

- Monitor Agent automates the full lifecycle — upload, deploy, install, validate, cleanup
- Includes an LLM-powered diagnostician for auto-repair
- Assertions validate deployment outputs on completion
- Detached mode runs long operations in the background via the daemon
- Use `--keep` to retain a deployment on failure for manual inspection
